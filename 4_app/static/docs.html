Memory System

MediAssist has two layers of memory for personalised, coherent conversations.

ğŸ’¬

Short-term (context window)

Pass recent turns in the history field. The model reads the full thread â€” follow-up messages like "it's been 3 days" understand prior context.

ğŸ§ 

Long-term (vector memory)

Stable facts â€” age, allergies, chronic conditions â€” are automatically extracted and stored in ChromaDB. Retrieved on future queries across sessions.

Code Examples

Complete, copy-pasteable examples for common use cases.

Python â€” full conversation with memory

import requests

BASE = "http://localhost:8000"

# Create a session
session_id = requests.post(f"{BASE}/session").json()["session_id"]

history = []

def ask(text):
    resp = requests.post(f"{BASE}/query", json={
        "text":       text,
        "session_id": session_id,
        "history":    history,
    }).json()

    # Store turn in history for next message
    history.append({"role": "user",      "content": text})
    history.append({"role": "assistant", "content": resp["answer"]})
    if len(history) > 12:
        history[:] = history[-12:]  # keep last 6 exchanges

    return resp

# Conversation
r = ask("I'm 34 and allergic to penicillin. I have a sore throat.")
print(f"Triage: {r['triage_level']}")  # LOW or MEDIUM
print(r["answer"])

r2 = ask("The fever started this morning â€” 38.9Â°C")
print(f"Triage: {r2['triage_level']}")  # likely MEDIUM now
print(r2["answer"])  # model knows about penicillin allergy from memory

JavaScript (fetch)

async function askMediAssist(text, sessionId = null, history = []) {
  const res = await fetch("http://localhost:8000/query", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ text, session_id: sessionId, history }),
  });
  return res.json();
}

// Usage
